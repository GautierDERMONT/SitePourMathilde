Diagnostic EmailJS

Résolution des problèmes d'envoi d'emails

Scénario 1: Rien dans l'Email History

Si vous ne voyez aucune entrée dans l'Email History d'EmailJS :

  • Les emails ne sont pas envoyés du tout par le service
  • Il y a probablement une erreur JavaScript qui empêche l'envoi
  • Le formulaire n'arrive pas à communiquer avec EmailJS

Solutions :

  • Ouvrez la console développeur (F12) pour voir les erreurs JavaScript
  • Vérifiez que les IDs de service et template sont corrects
  • Testez avec le bouton ci-dessous pour diagnostiquer la connexion à EmailJS

Scénario 2: Statut "Failed" dans Email History

Si vous voyez des entrées avec statut "Failed" :

  • EmailJS reçoit la requête mais ne peut pas envoyer l'email
  • Problème de configuration du service email
  • Template mal configuré ou variables manquantes

Solutions :

  • Vérifiez la configuration de votre service email dans EmailJS
  • Assurez-vous que votre template utilise les bonnes variables
  • Vérifiez que l'adresse email de destination est correcte

Test d'envoi d'email

Utilisez ce formulaire pour tester l'envoi d'email avec EmailJS :

Vérification de configuration

Assurez-vous que votre code JavaScript contient les bons identifiants :

emailjs.init("vkBvpfM1lMN-iC0kR"); // User ID
emailjs.sendForm('service_zs48k1a', 'template_fm831la', this); // Service ID et Template ID

Vérifiez que ces identifiants correspondent exactement à ceux de votre compte EmailJS.
